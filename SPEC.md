# 智能客服 Agent 规格说明

## 1. 项目概述

- **项目名称**: 智能客服 Agent (Smart Customer Service Agent)
- **类型**: Web 应用 (前后端分离)
- **核心功能**: 提供智能客服对话服务，支持多轮对话、FAQ 检索、意图识别、自动转人工
- **目标用户**: 网站访客、企业管理员

## 2. 技术架构

### 前端
- 纯 HTML + CSS + JavaScript (单页应用)
- LocalStorage 存储对话历史

### 后端
- Express.js + SQLite (轻量级，无需额外数据库)
- RESTful API 设计

## 3. 功能规格

### 3.1 客服对话界面 (/)

#### 用户端功能
- **多轮对话**: 支持连续对话，上下文保持
- **输入区域**: 文本输入框 + 发送按钮
- **消息气泡**: 用户消息右对齐(蓝色)，客服消息左对齐(灰色)
- **欢迎语**: 首次打开显示欢迎消息和快捷问题按钮
- **快捷问题**: 点击常见问题快速提问
- **转人工按钮**: 用户可主动请求人工客服
- **满意度评价**: 对话结束后可评价 (满意/不满意)

#### 意图识别
自动识别用户意图类别:
1. **退款** (refund) - 包含"退款"、"退货"、"钱"、"款"等关键词
2. **查询订单** (order) - 包含"订单"、"物流"、"发货"、"快递"等关键词
3. **技术支持** (tech) - 包含"无法登录"、"报错"、"bug"、"问题"等关键词
4. **其他** (other) - 未能识别的意图

### 3.2 FAQ 知识库

预设常见问题:
- 如何申请退款？
- 订单什么时候发货？
- 如何查看物流信息？
- 账号无法登录怎么办？
- 商品质量问题如何处理？
- 如何修改收货地址？
- 优惠券如何使用？
- 联系客服电话是多少？

### 3.3 自动转人工

触发条件:
- 用户点击"转人工"按钮
- 用户连续 3 次未得到有效回答
- 用户明确要求人工服务

转人工后:
- 显示"正在转接人工客服，请稍候..."
- 模拟人工接入 (显示人工客服消息)

### 3.4 对话历史持久化

存储内容:
- 会话 ID
- 用户 ID (匿名)
- 对话消息 (时间、角色、内容)
- 意图识别结果
- 满意度评价
- 会话状态 (进行中/已结束)

### 3.5 管理后台 (/admin)

#### 功能
- **对话记录**: 列表展示所有会话，支持搜索
- **满意度统计**: 显示满意/不满意比例
- **数据导出**: 导出对话记录为 JSON

#### 界面
- 顶部统计卡片: 总会话数、已结束数、满意度
- 对话列表: 会话ID、时间、用户意图、状态、评价
- 点击查看详情: 完整对话内容

## 4. 视觉设计

### 配色方案
- 主色: #4F46E5 (靛蓝色)
- 辅助色: #10B981 (绿色)
- 背景: #F9FAFB
- 用户消息: #4F46E5
- 客服消息: #E5E7EB
- 文字: #1F2937

### 布局
- 客服界面: 右侧固定聊天气泡式对话框
- 管理后台: 左侧导航 + 右侧内容区

### 字体
- 思源黑体 / system-ui

## 5. API 接口

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/chat | 发送消息，获取回复 |
| GET | /api/sessions | 获取会话列表 (管理后台) |
| GET | /api/sessions/:id | 获取会话详情 |
| POST | /api/sessions/:id/feedback | 提交满意度评价 |
| GET | /api/stats | 获取统计数据 |

## 6. 验收标准

- [ ] 用户可以发送消息并获得智能回复
- [ ] 系统能正确识别退款、订单、技术支持意图
- [ ] 连续3次未能回答问题自动转人工
- [ ] 对话历史存储到服务器
- [ ] 管理后台可以查看所有对话记录
- [ ] 管理后台显示满意度统计
- [ ] 用户可以主动点击转人工
- [ ] 对话结束后可以评价满意度
